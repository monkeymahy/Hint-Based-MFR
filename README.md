# Hint-Based MFR

这是一个基于 Python OCC 的 STEP 模型特征识别程序。它参考论文
"Hint-based generic shape feature recognition from three-dimensional B-rep models"
中的思路，用零件的几何和拓扑关系识别三类制造特征：

- `1 hole`：孔
- `2 boss`：凸台
- `3 chamfer`：倒角

## 识别思路

程序不会只看单个面的形状，而是先把整个零件拆成一个“面之间的关系图”。每个面会记录：

- 面类型：平面、圆柱面、圆锥面等。
- 面积、中心位置、法向方向。
- 圆柱面或圆锥面的轴线方向和轴线位置。
- 圆柱面的 U 向覆盖范围。
- 圆边、完整圆边、直线边数量。
- 相邻面关系，以及哪些相邻关系来自内环。

当前识别顺序是：

1. 先识别由内环引出的孔。
2. 再识别倒角。
3. 再识别被倒角挡住入口的孔。
4. 再识别被 STEP 拆成多段但合起来闭合一圈的同轴圆柱孔壁。
5. 再用局部圆柱面规则补充识别漏掉的孔。
6. 最后用结构化规则统一识别凸台。

已经识别出的面不会被后续规则覆盖。凸台不再由“某个面是 outward 圆柱面”这种类型规则给出，而是由一个独立的结构化 pass 统一识别，下面单列说明。

## 孔的识别规则

孔的本质是：从零件中移除了一段圆柱形体积。因此当前程序只把圆柱形孔识别为 `hole`，不会把多边形孔或纯平面侧壁的开口识别为孔。

一个区域会被识别为孔，通常需要满足这些条件：

- 有圆柱面作为孔壁。
- 孔壁的径向方向是 inward，即向零件内部凹进去。
- 如果孔壁被拆成多个圆柱面，这些圆柱面必须共用同一根轴线。
- 孔壁必须形成完整圆形结构，不能是几个不相关的局部圆角。
- 孔结构中不能混入和孔无关的平面、圆角或其他零件表面。
- 当完整圆孔在 STEP 中被拆成多个半圆柱或局部圆柱面时，程序会把直接相邻、同轴、同为 inward 的圆柱面成组；只有组合后的 U 向角度覆盖接近 `2π`，并且周围有内环 carrier、盲孔底面或有效切割边界证据时，才会识别为 `hole`。

### 完整圆柱孔壁

程序现在用两种方式判断圆柱孔壁是否完整：

- 圆柱面的 U 向覆盖范围接近 `2π`。
- 或者该圆柱面虽然有局部缺口，但仍有至少两条完整圆边。

第二条用于处理“U 形局部缺口”的孔：缺口没有贯穿整个圆柱面时，圆柱面上下仍保留完整圆边，因此仍然可以作为完整孔壁识别。相反，贯穿缺口、1/4 圆柱圆角、半圆槽等通常没有两条完整圆边，不会因为这条规则被误识别为孔。

对于“像一根被斜劈掉的竹子，中间仍有圆孔”的情况，STEP 里通常仍会把孔壁表达为圆柱面，只是这个圆柱面被一个斜平面裁剪，边界上会出现圆/椭圆类曲线。程序会允许这种斜切平面作为孔的边界条件：圆柱孔壁仍可识别为 `hole`，但斜切出来的外表面本身不会被包含进 `hole`。

对于被斜切或多面切割后拆成两段半圆柱的孔，程序不要求单个圆柱面自己完整成环，而是要求同一组圆柱面直接邻接、共轴、方向一致，并且合计覆盖完整圆周。这样可以识别“被斜劈竹子中间的圆孔”，同时避免把几个相隔较远或轴线不同的 1/4 圆柱圆角拼成孔。

### 孔包含哪些面

会被包含进 `hole` 的面：

- 孔的圆柱侧壁。
- 与孔壁同轴、同为 inward 的圆柱侧壁。
- 盲孔底面。
- 阶梯孔中间的圆形台阶面，但前提是它还连接另一个同轴 inward 孔壁。

不会被包含进 `hole` 的面：

- 孔入口所在的大平面。
- 与孔相连但属于零件外壳的普通外表面。
- 只是在拓扑上邻接孔、但没有下沉到孔内部的平面。

盲孔底面和外表面的关键区别是位置：盲孔底面必须相对入口 carrier 有可测的下沉距离。即使某个面也是外壳的一部分，只要它没有相对孔入口下沉，就不能作为孔底面加入 `hole`。

如果孔口有倒角，程序会先识别倒角，再检查倒角后面是否连接完整 inward 圆柱孔壁。如果该锥形倒角把孔壁连接到一个对齐的内环 carrier，倒角后面的孔壁也会被识别为 `hole`。

## 凸台的识别规则

凸台的本质是：从某个基准表面向外凸出的闭合形状。早期的类型化规则（“outward 圆柱面就是凸台”“一圈平面侧壁就是平面凸台”）会漏掉没有顶面的凸台、没有闭合侧壁的凸台，也会把结构上完全一样的凹槽误判成凸台。当前实现改用一个**结构化 pass** 统一识别凸台，不再区分圆柱凸台和平面凸台。

### 结构化定义

一个凸台实例必须同时具备三部分，以这个结构作为载体来识别：

1. **闭合侧壁环**：一圈侧壁面把一个开口围起来，并且这一圈不会“漏”到与凸台无关的外表面上。侧壁可以是圆柱面，也可以是平面，混合也可以。
2. **顶面**：一个盖住侧壁环的顶面。顶面可以直接贴在侧壁上；如果顶面和侧壁之间隔着过渡面（倒角/圆角），那么“过渡面 + 顶面”一起盖住侧壁环也可以。
3. **底面**：凸台所凸出的基准表面。底面可以是任何面，只用来确定凸出方向，本身不进入凸台。

三部分缺一不可：只有侧壁没有顶面、或侧壁不闭合的结构，都不算凸台。

### 凸台与凹槽的区分

凸台和凹槽在结构上完全对称（都是一圈侧壁 + 顶 + 底），唯一能区分的是凸出方向：

- 圆柱侧壁用径向方向区分：`outward`（径向 > `0.2`）的是凸台，`inward` 的是孔/凹槽。
- 平面侧壁没有径向方向，改用相对底面法向的有符号偏移区分：侧壁中心落在底面法向正侧的是凸台，落在负侧的是凹槽。

### 种子与基准面

pass 从一个侧壁种子出发找凸台。种子有两种：

- **圆柱种子**：径向 `outward` 的圆柱面。这是无歧义的凸台种子。
- **平面种子**：被某个内环 carrier 持有的平面侧壁，且该平面相对 carrier 凸出（有符号偏移为正）。只有能找到内环 carrier 的平面侧壁才作为种子，避免把凹槽的平面侧壁当种子。

找到种子后确定基准面（carrier），分三种情形：

- **情形 A（内环 carrier）**：种子被某个带内环的面持有，那个面就是基准面。这是最经典的 hint：凸台从基准面上的一个孔里凸出来。
- **情形 B（外壁）**：圆柱侧壁通过基准面的外环连接（而不是内环）。这说明它是零件外壳本身的圆柱壁，不是凸出物，直接拒绝。
- **情形 C（独立圆柱/装配）**：圆柱两端都是 cap、没有拓扑相连的基准面。这时做几何搜索：在与 cap 共面、法向对齐的所有平面里，找一个外边界**严格包含** cap 连接圆的面作为基准面。底面可能在装配体的另一个 solid 上，所以是几何搜索而非拓扑搜索。包含判定是形状无关的：把 cap 外边界采样成点，要求每个点都落在候选底面的外环多边形内部，且至少有一个点严格落在内部（防止把同尺寸重合面当成更大的底面）。两端都找不到更大的底面时，这是一根真正的独立圆销，不是凸台，拒绝。

### 侧壁环生长

从种子出发做 BFS 扩展侧壁环。一个相邻面能加入环，需要：

- 相对 carrier 沿凸出方向凸出（和种子同向）。
- 是 `outward` 圆柱面（且与已有圆柱侧壁共轴），或与 carrier 法向近似正交的平面。
- 不带内环、不是 carrier 本身。

如果遇到一个过渡面（倒角/圆角，即“桥”），桥本身不加入环、保持自己的标签，但允许穿过桥去认领桥另一侧同轴的圆柱侧壁段。这样多级凸台（中间被锥台/圆角分成几段）也能合成一个环。

### 闭合性检查

环要闭合：环里每个面的每个非侧壁邻居，必须是 carrier、通向顶面的过渡面、或环内另一个面。带内环的邻居只有当它正好是盖住环的顶面时才算合法边界，否则视为环漏到了外表面上，拒绝这个凸台。

### 顶面与边界约束

顶面必须盖住侧壁环，且有两个约束：

- **位置约束**：顶面是与 carrier 法向对齐的平面，且顶面中心沿凸出方向的偏移必须严格大于所有侧壁环面（顶面在侧壁外侧，而不是和侧壁齐平或更靠内）。顶面可以带内环（孔穿过顶面），仍然合法。
- **边界约束（不能超过侧壁）**：顶面 + 过渡面组成的“盖子”外边界最多与侧壁顶边重合，不能超出。实现上检查盖子里每个面的每个邻居：必须是侧壁环面、carrier、盖子内另一个面、或穿过顶面的孔壁；只要碰到任何一个与凸台结构无关的外表面，就说明顶面溢出了侧壁边界，拒绝这个凸台。

满足以上全部检查后，侧壁环 + 顶面 + 过渡面合成一个 `boss` 实例；carrier/底面不进入。

## 倒角的识别规则

倒角的本质是：两个非共面的正常表面之间，有一个较小的过渡面。这个过渡面可以是平面，也可以是锥面。

一个面会被识别为倒角，通常需要满足这些条件：

- 它夹在至少两个支撑面之间。
- 它和支撑面之间的夹角符合倒角的常见角度范围。
- 它相对于支撑面足够小，不能是普通的大平面。
- 它不能带有内环关系，因为带内环的面更可能是孔或凸台的 carrier。

平面倒角主要检查“斜面 + 两侧支撑面 + 面积较小”。锥形倒角主要检查“锥面 + 过渡性质 + 邻接支撑面”，并要求拓扑足够简单：邻接面数量和边数量不能过多，且至少要连接一个平面支撑面。复杂锥面、孔/槽内部的大锥形侧壁不会因为是锥面而自动标成 `chamfer`。

特别规则：两个曲面之间的连接面不算倒角。例如圆柱面和另一个曲面之间的圆滑连接，即使看起来也是过渡区域，也不标记为 `chamfer`。普通的平面-平面倒角、平面-圆柱倒角仍然可以识别。

## 参数取值说明

下面这些参数是当前实现中直接参与判断的默认值：

| 参数 | 当前值 | 作用 |
| --- | ---: | --- |
| `radial_threshold` | `0.2` | 判断圆柱/圆锥面的径向方向。小于 `-0.2` 认为是 inward，大于 `0.2` 认为是 outward。 |
| `axis_alignment_threshold` | `0.7` | 判断轴线和法向是否对齐，使用单位向量点积绝对值。 |
| `axis_distance_tolerance_ratio` | `1.0e-5` | 判断多个圆柱是否共轴，按模型包围盒对角线缩放。 |
| `full_cylinder_u_tolerance` | `1.0e-3` rad | 判断圆柱 U 向跨度是否接近完整一圈。 |
| `hole_angular_coverage_tolerance` | `0.05` rad | 判断多个圆柱孔壁合起来是否闭合成完整圆。 |
| `chamfer_min_angle` | `18.0°` | 平面倒角与支撑面之间的最小锐角。 |
| `chamfer_max_angle` | `72.0°` | 平面倒角与支撑面之间的最大锐角。 |
| `chamfer_max_support_area_ratio` | `0.35` | 平面倒角面积相对支撑面的最大比例。 |

其他固定辅助规则：

- 内环识别出的局部组件如果超过 `32` 个面，会认为这个 hint 太大、不够局部，直接跳过。
- 圆柱孔壁可以通过 U 向跨度接近 `2π` 判定为完整，也可以通过至少两条完整圆边判定为完整。
- 被斜平面裁剪的圆柱孔壁可以通过“完整 inward 圆柱面 + 带圆/椭圆边的斜切平面边界”识别；斜切平面只作为边界，不加入 hole。
- 孔中的锥形倒角要求与孔壁共轴，并且径向值满足 `0.2 < abs(radial) < 0.95`。
- 普通锥形倒角要求 `0.2 < abs(radial) < 0.995`，同时不能只是两个曲面之间的连接面；邻接面数量需要不超过 `6`，边数量需要不超过 `8`，并且至少有一个平面支撑面。
- hole 的平面 cap 如果有明确入口 carrier，必须相对该 carrier 有下沉距离，避免把孔连接的外表面误收进 hole。
- 简单圆形 cap 的邻接面数量不能超过 `3`，且 cap 面积通常需要小于圆柱侧壁面积的 `65%`。
- 一般圆形端面加入特征时，面积需要小于对应侧壁面积的 `125%`；hole 的内环 carrier 端面加入时更严格，需要不超过侧壁面积的 `55%`。
- 平面倒角如果面积较小，使用 `max(中位面面积 * 4.0, 模型包围盒对角线^2 * 0.025)` 作为普通面积上限；较大的长倒角还需要额外支撑面法向检查，且面积不能超过 `max(中位面面积 * 8.0, 模型包围盒对角线^2 * 0.08)`。
- 凸台结构化 pass 中，平面侧壁法向与 carrier 法向的绝对点积需要小于等于 `0.35`（正交），顶面法向与 carrier 法向的绝对点积需要大于等于 `axis_alignment_threshold`（`0.7`，对齐）。
- 凸台侧壁是否真的向外凸出，用面中心相对 carrier 的有符号偏移判断，容差为 `max(模型包围盒对角线 * 1.0e-7, 1.0e-7)`；偏移符号由种子固定，环内所有侧壁必须与种子同向。
- 凸台顶面位置约束：顶面中心沿凸出方向的偏移必须严格大于所有侧壁环面的偏移（同样用上述容差）。
- 凸台底面边界包含判定（情形 C）的共面容差为 `max(模型包围盒对角线 * 1.0e-6, 1.0e-6)`；底面外环多边形必须包含 cap 外边界全部采样点，且至少一个点严格落在内部。

## 运行

使用 `conda run`，确保 OpenCASCADE 的 DLL 路径被正确激活。

`recognizer.py` 内部使用扁平导入（`from geometry import ...`），所以需要从 `mfr_recognizer` 包目录内运行，否则 `python -m mfr_recognizer.*` 会找不到 `geometry` 模块。

单文件识别（在 `mfr_recognizer/` 目录内）：

```powershell
cd mfr_recognizer
F:\miniforge\Scripts\conda.exe run -n mfr python -m cli ..\data\step\01010028.step --verbose
```

评估样本数据集（包目录内直接跑评估脚本，或用根目录的 `evaluate.py` 入口）：

```powershell
cd mfr_recognizer
F:\miniforge\Scripts\conda.exe run -n mfr python evaluate.py --dataset ..\data --details
```

批量生成预测标签：

```powershell
cd mfr_recognizer
F:\miniforge\Scripts\conda.exe run -n mfr python batch_predict.py --step-dir ..\data\step --out-dir ..\data\pred_label --overwrite
```

> 注：`python -m mfr_recognizer.cli` 这种从仓库根目录的模块调用方式因扁平导入暂时不可用，请在 `mfr_recognizer/` 目录内以脚本方式运行。
