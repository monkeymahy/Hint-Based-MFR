# Hint-Based MFR

这是一个基于 Python OCC 的 STEP 模型特征识别程序。它参考论文
"Hint-based generic shape feature recognition from three-dimensional B-rep models"
中的思路，用零件的几何和拓扑关系识别三类制造特征：

- `1 hole`：孔
- `2 boss`：凸台
- `3 chamfer`：倒角

## 识别思路

程序不会只看单个面的形状，而是先把整个零件拆成一个“面之间的关系图”。每个面会记录：

- 面类型：平面、圆柱面、圆锥面等。
- 面积、中心位置、法向方向。
- 圆柱面或圆锥面的轴线方向和轴线位置。
- 圆柱面的 U 向覆盖范围。
- 圆边、完整圆边、直线边数量。
- 相邻面关系，以及哪些相邻关系来自内环。

当前识别顺序是：

1. 先用一条基于定义的 pass 统一识别孔（枚举 inward 圆柱侧壁 → 分组 → 判定完整 2π 圆周 → 附加盲孔底面 → 合并阶梯孔）。
2. 再识别倒角。
3. 最后用结构化规则统一识别凸台。

已经识别出的面不会被后续规则覆盖。凸台不再由“某个面是 outward 圆柱面”这种类型规则给出，而是由一个独立的结构化 pass 统一识别，下面单列说明。

## 孔的识别规则

孔的定义是：从零件中移除了一段圆柱形体积（`docs/definition.md`）。这个定义的可检特征是**完整的 2π 圆柱侧壁**——一圈没有断开的管壁。用户给出的三条实际口径：

- 侧壁被 STEP 拆开、**拓扑上不连接**的 → 不是孔；
- 侧壁被切/被修饰、但**仍有完整 2π 圆周不断** → 是孔；
- 盲孔底面必须**与这条完整圆周相连**、且**在圆周内部有面积**；底面自身可以有内环孔，不必铺满。

程序不再区分“内环孔 / 倒角桥接孔 / split 半圆孔 / 兜底孔”，而是用一条 wall-first pass 统一识别。多边形孔、纯平面侧壁开口继续不算孔。

### 单 pass 流程

1. **枚举 inward 圆柱**：所有径向值 `< -radial_threshold` 且带轴向/半径信息的圆柱面。
2. **按拓扑 + 共轴 + 同半径分组**：只沿共享 edge 做 BFS；两个面属于同一组，当且仅当**它们直接共享 edge** 并且轴向、轴线位置、半径都匹配（`_coaxial_same_radius`）。几何上共轴但拓扑不相连的两段半圆 → 不同组（对应“拓扑上分开 → 不是孔”）。
3. **闭合性检查**：组内所有面的 `u_span` 之和 `≥ 2π - hole_angular_coverage_tolerance`。任一半圆缺角、U 形槽、1/4 圆角都在这一步被排除。
4. **hint 校验（carrier）**：组必须能找到一个 hint 锚点，才算真正的孔而不是斜切穿过实体的圆柱片段。允许的锚点有两种：
   - **直接 carrier**：某个 wall 出现在其邻居面的 inner-loop 邻接中（Li et al. 的经典 hint：wall 位于 carrier 的内环里）。
   - **过渡桥 carrier**：wall 的邻居是 cone 或 torus 过渡面，而这个过渡面又落在某个 carrier 的 inner-loop 邻接中——即倒角/圆角挡住了入口，carrier 隔一层。

   没有 carrier 的 wall 组仍然可以是孔（保守起见，wall 本身自带证据），但**不允许附加盲孔底面**——一个拿不到 carrier 的圆柱片段最典型的成因是“斜平面切穿实体的圆柱截面”，把邻接平面当底面会漏进外壳。
5. **盲孔底面**（仅在 carrier 命中时启用，`_find_blind_bottoms`）：候选是与 wall 组共享 edge 的平面邻居。对每个候选平面 P：
   - 只考察它与 wall 共享的、类型为**完整/局部圆弧**的边，且圆弧半径与 wall 半径一致；
   - 把这些圆弧按“沿轴投影”分桶（一个桶代表一个轴向 z 位置的“嘴”）；
   - 桶内累计圆弧参数跨度 `≥ 2π - hole_angular_coverage_tolerance`，且这些边**位于 P 的外环**（wall 相对 P 是外环邻居，说明 P 沉在孔内部，而不是 mouth carrier）→ P 是这个位置的 blind bottom。
6. **合并阶梯孔（`_merge_stepped_holes`）**：若两个已确认的 hole feature 共享一个 shelf plane（它是一个 feature 的 blind bottom、同时是另一个 feature 的 carrier 邻接平面），且两个 feature 同轴，则通过 union-find 合并为一个 hole 实例，shelf plane 一并纳入。

### 孔包含哪些面

- 侧壁：wall 组内所有 inward 圆柱面。
- 盲孔底面：`_find_blind_bottoms` 判定通过的平面。
- 阶梯孔中间的 shelf plane：`_merge_stepped_holes` 合并后进入。

不进入孔的面：入口大平面（那是 mouth carrier）、桥接的 cone/torus 过渡面（保留原本 label；cone 通常留在 `other`）、以及所有和这个 wall 组无关的邻接面。


## 凸台的识别规则

凸台的本质是：从某个基准表面向外凸出的闭合形状。早期的类型化规则（“outward 圆柱面就是凸台”“一圈平面侧壁就是平面凸台”）会漏掉没有顶面的凸台、没有闭合侧壁的凸台，也会把结构上完全一样的凹槽误判成凸台。当前实现改用一个**结构化 pass** 统一识别凸台，不再区分圆柱凸台和平面凸台。

### 结构化定义

一个凸台实例必须同时具备三部分，以这个结构作为载体来识别：

1. **闭合侧壁环**：一圈侧壁面把一个开口围起来，并且这一圈不会“漏”到与凸台无关的外表面上。侧壁可以是圆柱面、平面、或自由曲面，混合也可以。侧壁也可以带内环（例如环形侧壁：外边界是凸台轮廓、内边界是穿台小孔）；这种侧壁的内环邻居是穿台孔的壁，属于凸台结构的一部分。
2. **顶面**：一个盖住侧壁环的顶面。顶面可以直接贴在侧壁上；如果顶面和侧壁之间隔着过渡面（倒角/圆角），那么“过渡面 + 顶面”一起盖住侧壁环也可以。
3. **底面**：凸台所凸出的基准表面。底面可以是任何面，只用来确定凸出方向，本身不进入凸台。

三部分缺一不可：只有侧壁没有顶面、或侧壁不闭合的结构，都不算凸台。

### 凸台与凹槽的区分

凸台和凹槽在结构上完全对称（都是一圈侧壁 + 顶 + 底），唯一能区分的是凸出方向：

- 圆柱侧壁用径向方向区分：`outward`（径向 > `0.2`）的是凸台，`inward` 的是孔/凹槽。
- 平面侧壁和自由曲面侧壁没有径向方向，改用相对底面法向的有符号偏移区分：侧壁中心落在底面法向正侧的是凸台，落在负侧的是凹槽。

### 种子与基准面

pass 从一个侧壁种子出发找凸台。种子有两种：

- **圆柱种子**：径向 `outward` 的圆柱面。这是无歧义的凸台种子。
- **平面种子**：被某个内环 carrier 持有的平面侧壁，且该平面相对 carrier 凸出（有符号偏移为正）。只有能找到内环 carrier 的平面侧壁才作为种子，避免把凹槽的平面侧壁当种子。

找到种子后确定基准面（carrier），分三种情形：

- **情形 A（内环 carrier）**：种子被某个带内环的面持有，那个面就是基准面。这是最经典的 hint：凸台从基准面上的一个孔里凸出来。
- **情形 A'（内环 carrier 被过渡面挡住）**：种子和 carrier 之间隔着一个圆角/圆锥过渡面时，carrier 的内环邻居是过渡面而不是侧壁本身，情形 A 会漏掉。允许跨过一个 cone/torus 过渡面去找 carrier：过渡面的邻居里若有带内环的面，即视为该侧壁的 carrier。
- **情形 B（外壁）**：圆柱侧壁通过基准面的外环连接（而不是内环）。这说明它是零件外壳本身的圆柱壁，不是凸出物，直接拒绝。
- **情形 C（独立圆柱/装配）**：圆柱两端都是 cap、没有拓扑相连的基准面。这时做几何搜索：在与 cap 共面、法向对齐的所有平面里，找一个外边界**严格包含** cap 连接圆的面作为基准面。底面可能在装配体的另一个 solid 上，所以是几何搜索而非拓扑搜索。包含判定是形状无关的：把 cap 外边界采样成点，要求每个点都落在候选底面的外环多边形内部，且至少有一个点严格落在内部（防止把同尺寸重合面当成更大的底面）。两端都找不到更大的底面时，这是一根真正的独立圆销，不是凸台，拒绝。

### 侧壁环生长

从种子出发做 BFS 扩展侧壁环。一个相邻面能加入环，需要：

- 相对 carrier 沿凸出方向凸出（和种子同向）。
- **横向性**（类型无关主判据）：面法向与 carrier 法向近似正交（绝对点积 ≤ `0.35`），即面朝向侧向、不是顶/底 cap。这对平面、自由曲面都成立。
- **类型特化**：圆柱侧壁额外要求 `outward` 径向且与已有圆柱侧壁共轴；圆锥面和圆环面只作过渡面（倒角/圆角），不当侧壁。
- 侧壁可以带内环（环形侧壁，内边界是穿台小孔）。为防止两个相邻凸台通过一个共享侧壁误并，带内环的侧壁要求其内环邻居必须是 `inward` 孔壁，不能是另一个 `outward` 片段。

如果遇到一个过渡面（倒角/圆角，即“桥”），桥本身不加入环、保持自己的标签，但允许穿过桥去认领桥另一侧同轴的圆柱侧壁段。这样多级凸台（中间被锥台/圆角分成几段）也能合成一个环。

### 闭合性检查

环要闭合：环里每个面的每个非侧壁邻居，必须是 carrier、通向顶面的过渡面、环内另一个面、或穿台孔壁。穿台孔壁指侧壁内环邻居、`inward` 圆柱壁、或已被识别为 `hole` 的面——这些是凸台结构内的合法边界。带内环的邻居只有当它正好是盖住环的顶面时才算合法边界，否则视为环漏到了外表面上，拒绝这个凸台。

### 顶面与边界约束

顶面必须盖住侧壁环，且有两个约束：

- **位置约束**：顶面是与 carrier 法向对齐的平面，且顶面中心沿凸出方向的偏移必须严格大于所有侧壁环面（顶面在侧壁外侧，而不是和侧壁齐平或更靠内）。顶面可以带内环（孔穿过顶面），仍然合法。
- **边界约束（不能超过侧壁）**：顶面 + 过渡面组成的“盖子”外边界最多与侧壁顶边重合，不能超出。实现上检查盖子里每个面的每个邻居：必须是侧壁环面、carrier、盖子内另一个面、或穿过顶面的孔壁；只要碰到任何一个与凸台结构无关的外表面，就说明顶面溢出了侧壁边界，拒绝这个凸台。

### 比例约束（周长/π > 高度）

即使侧壁环、顶面、边界都合格，还要求凸台**周长 / π > 高度**，否则拒绝。周长取顶面外边界的实测多边形长度（形状无关），高度取顶面沿 carrier 法向偏移与侧壁环最低面偏移之差的绝对值。对圆柱等价于"直径 > 高度"，用来拒绝细长销钉/圆棒那种结构上像凸台但工艺上不算凸台的情形。

满足以上全部检查后，侧壁环 + 顶面 + 过渡面合成一个 `boss` 实例；carrier/底面不进入。

## 倒角的识别规则

倒角的本质是：两个非共面的正常表面之间，有一个较小的**平面**过渡面(截面是平的)。按 AP224 定义，chamfer 是 "flat cross section" 过渡，因此程序只把平面过渡面识别为 `chamfer`；圆锥形过渡面(如孔口锥形倒角、圆锥形沉孔)不再作为 `chamfer` 输出，即使它在几何上起过渡作用。

一个面会被识别为倒角，通常需要满足这些条件：

- 它是平面。
- 它夹在至少两个支撑面之间。
- 它和支撑面之间的夹角符合倒角的常见角度范围。
- 它相对于支撑面足够小，不能是普通的大平面。
- 它不能带有内环关系，因为带内环的面更可能是孔或凸台的 carrier。

特别规则：两个曲面之间的连接面不算倒角。例如圆柱面和另一个曲面之间的圆滑连接，即使看起来也是过渡区域，也不标记为 `chamfer`。普通的平面-平面倒角、平面-圆柱倒角仍然可以识别。

注：孔口的锥形过渡面仍然被用作"倒角桥接孔"的证据(它把孔壁连接到内环 carrier)，只是它自己不再被标为 `chamfer`——那种情形下锥面通常留为 `other`，被它遮住的圆柱孔壁仍会被识别为 `hole`。

## 参数取值说明

下面这些参数是当前实现中直接参与判断的默认值：

| 参数 | 当前值 | 作用 |
| --- | ---: | --- |
| `radial_threshold` | `0.2` | 判断圆柱/圆锥面的径向方向。小于 `-0.2` 认为是 inward，大于 `0.2` 认为是 outward。 |
| `axis_alignment_threshold` | `0.7` | 判断轴线和法向是否对齐，使用单位向量点积绝对值。 |
| `axis_distance_tolerance_ratio` | `1.0e-5` | 判断多个圆柱是否共轴，按模型包围盒对角线缩放。 |
| `hole_angular_coverage_tolerance` | `0.05` rad | 判断多个圆柱孔壁合起来是否闭合成完整圆，以及盲孔底面圆弧累计是否闭合。 |
| `chamfer_min_angle` | `18.0°` | 平面倒角与支撑面之间的最小锐角。 |
| `chamfer_max_angle` | `72.0°` | 平面倒角与支撑面之间的最大锐角。 |
| `chamfer_max_support_area_ratio` | `0.35` | 平面倒角面积相对支撑面的最大比例。 |

其他固定辅助规则：

- 孔 pass 的 wall 分组容差：半径差 `≤ max(模型对角线 * 1.0e-5, 1.0e-6)`；共轴判定用轴向单位向量点积 `≥ 1 - axis_distance_tolerance_ratio`，轴线间垂直距离 `≤ max(模型对角线 * axis_distance_tolerance_ratio, 1.0e-7)`。
- 孔 pass 的过渡桥 carrier：只允许 wall 的直接邻居为 `cone` 或 `torus` 时跨一层，不做多跳。
- 孔 pass 的盲孔底面判定沿轴分桶：共享圆弧按 `dot(center, axis)` 保留 6 位小数分桶；同桶累计参数跨度 `≥ 2π - hole_angular_coverage_tolerance` 才算“一张嘴”。
- 平面倒角如果面积较小，使用 `max(中位面面积 * 4.0, 模型包围盒对角线^2 * 0.025)` 作为普通面积上限；较大的长倒角还需要额外支撑面法向检查，且面积不能超过 `max(中位面面积 * 8.0, 模型包围盒对角线^2 * 0.08)`。
- 凸台结构化 pass 中，侧壁法向与 carrier 法向的绝对点积需要小于等于 `0.35`（横向/正交，对平面、自由曲面统一适用），顶面法向与 carrier 法向的绝对点积需要大于等于 `axis_alignment_threshold`（`0.7`，对齐）。
- 凸台侧壁是否真的向外凸出，用面中心相对 carrier 的有符号偏移判断，容差为 `max(模型包围盒对角线 * 1.0e-7, 1.0e-7)`；偏移符号由种子固定，环内所有侧壁必须与种子同向。
- 凸台顶面位置约束：顶面中心沿凸出方向的偏移必须严格大于所有侧壁环面的偏移（同样用上述容差）。
- 凸台底面边界包含判定（情形 C）的共面容差为 `max(模型包围盒对角线 * 1.0e-6, 1.0e-6)`；底面外环多边形必须包含 cap 外边界全部采样点，且至少一个点严格落在内部。
- 凸台 carrier 可达性（情形 A'）允许跨过一个 cone/torus 过渡面：过渡面本身不作为 carrier，但如果它的邻居中有带内环的面，就以那个面为 carrier。
- 凸台比例约束：顶面外边界周长 / π > 顶面到侧壁环最低面沿 carrier 法向的距离绝对值，否则拒绝。周长取顶面外环实测多边形长度。

## 运行

使用 `conda run`，确保 OpenCASCADE 的 DLL 路径被正确激活。

`recognizer.py` 内部使用扁平导入（`from geometry import ...`），所以需要从 `mfr_recognizer` 包目录内运行，否则 `python -m mfr_recognizer.*` 会找不到 `geometry` 模块。

单文件识别（在 `mfr_recognizer/` 目录内）：

```powershell
cd mfr_recognizer
F:\miniforge\Scripts\conda.exe run -n mfr python -m cli ..\data\step\01010028.step --verbose
```

评估样本数据集（包目录内直接跑评估脚本，或用根目录的 `evaluate.py` 入口）：

```powershell
cd mfr_recognizer
F:\miniforge\Scripts\conda.exe run -n mfr python evaluate.py --dataset ..\data --details
```

批量生成预测标签：

```powershell
cd mfr_recognizer
F:\miniforge\Scripts\conda.exe run -n mfr python batch_predict.py --step-dir ..\data\step --out-dir ..\data\pred_label --overwrite
```

> 注：`python -m mfr_recognizer.cli` 这种从仓库根目录的模块调用方式因扁平导入暂时不可用，请在 `mfr_recognizer/` 目录内以脚本方式运行。
